# Semantic Code Knowledge Graph — VS Code Extension

A local, continuously-updated knowledge graph of your codebase exposed to AI coding assistants via MCP.

---

## Features

| Feature | Description |
|---------|-------------|
| **Auto-indexing** | Parses your workspace on open and incrementally on save |
| **Knowledge Graph** | SQLite-backed graph of functions, classes, methods, files and their relationships |
| **Semantic Search** | 384-dim embeddings via `Xenova/all-MiniLM-L6-v2` (ONNX, no Python) |
| **MCP Server** | Exposes 4 tools to any MCP-compatible AI agent (Cursor, Claude, etc.) |
| **Graph Explorer** | Interactive D3 force-directed graph in the VS Code sidebar |
| **Context Inspector** | Real-time view of what the AI is querying |
| **Auto Docstrings** | Missing docstrings generated by your configured LLM (stored in DB only) |
| **Commit Messages** | Semantic graph diff → conventional commit message |

---

## MCP Tools

| Tool | What it does |
|------|-------------|
| `explore_symbol` | Returns a node's intent, signature, file path, and 1-hop callers/callees |
| `search_intent` | Semantic natural language search across all symbols |
| `get_blast_radius` | BFS via recursive SQL CTE — all dependents of a symbol |
| `suggest_commit_message` | Structural graph diff of staged changes → commit message |

---

## Extension Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `semanticKG.llmProvider` | `anthropic` | `anthropic` / `openai` / `ollama` |
| `semanticKG.llmApiKey` | `""` | API key for docstring + commit generation |
| `semanticKG.llmModel` | `claude-sonnet-4-20250514` | Model name |
| `semanticKG.mcpPort` | `3579` | Port for the local MCP HTTP server |
| `semanticKG.autoRebuildOnSave` | `true` | Incremental update on file save |
| `semanticKG.excludePatterns` | `["**/node_modules/**", ...]` | Glob patterns to skip |

## Architecture

```
VS Code Extension (Node.js)
│
├── FileParser          (web-tree-sitter WASM)
│   └── Extracts nodes: functions, classes, methods, interfaces, files
│
├── GraphStore          (better-sqlite3 + WAL mode)
│   ├── nodes table     (id, type, name, file_path, signature, docstring, checksum)
│   ├── edges table     (source_id, target_id, relation_type)
│   ├── embeddings table (node_id, vector BLOB)
│   └── Recursive CTE for BFS blast-radius queries
│
├── EmbeddingEngine     (@xenova/transformers, ONNX)
│   └── all-MiniLM-L6-v2, 384-dim, cosine similarity
│
├── MCPServer           (HTTP on localhost:3579)
│   └── 4 tools: explore_symbol, search_intent, get_blast_radius, suggest_commit_message
│
├── CommitDiffEngine    (git integration via VS Code SCM API)
│   └── Structural graph diff → LLM prompt → conventional commit
│
└── Webview Panels
    ├── GraphExplorer   (D3 force-directed graph)
    └── ContextInspector (real-time MCP query log)
```

## Connecting an AI Agent

Point your MCP-compatible AI to:

```
http://localhost:3579/mcp
```

To get the tool manifest:
```
GET http://localhost:3579/mcp/manifest
```

Example call:
```json
POST http://localhost:3579/mcp
{
  "tool": "search_intent",
  "params": { "query": "database connection handling", "limit": 5 }
}
```

### Cursor AI
In `.cursor/mcp.json`:
```json
{
  "mcpServers": {
    "semantic-kg": {
      "url": "http://localhost:3579/mcp"
    }
  }
}
```

### Claude Desktop
In `claude_desktop_config.json`:
```json
{
  "mcpServers": {
    "semantic-kg": {
      "command": "node",
      "args": [],
      "env": {},
      "url": "http://localhost:3579/mcp"
    }
  }
}
```
